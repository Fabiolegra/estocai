1 - Cadastro de Usuário
1.1 -
Implemente o módulo de Cadastro de usuários. O sistema deve permitir que novos usuários se cadastrem informando nome, e-mail e senha. O e-mail deve ser único no banco de dados. As senhas devem ser armazenadas de forma segura usando password_hash. Após o cadastro, o usuário deve ser redirecionado para a página de login
1.2 -
Crie o design da página de Cadastro de Usuário com uma abordagem mobile-first. O formulário deve ser vertical, com campos de nome, e-mail e senha ocupando a largura total em telas pequenas. Use placeholders claros e labels visíveis. 

2 - Autenticação de Usuários
2.1
Implemente a funcionalidade de login de usuário. O sistema deve validar e-mail e senha. Em caso de sucesso, crie uma sessão para o usuário e redirecione-o para o dashboard. Em caso de falha, exiba uma mensagem de erro na página de login.
2.2
Desenvolva a interface da página de Login, seguindo o princípio mobile-first. O formulário deve ser simples e centrado, contendo campos para `e-mail` e `senha`, e um botão de 'Entrar' bem destacado. Adicione um link para a página de cadastro. Mensagens de erro devem aparecer próximas aos campos ou em um alerta visível, usando a cor vermelha para indicar falha.

3 - dashboard
3.1 -
Crie a página do dashboard para usuários autenticados. Esta página deve apresentar uma visão geral do estoque, exibindo estatísticas como total de produtos, produtos com estoque crítico, etc. Inclua também atalhos para as principais operações de produtos
3.2 -
Projete o Dashboard com uma abordagem mobile-first. Em telas pequenas, os cards de estatísticas (total de produtos, estoque crítico, etc.) devem ser empilhados verticalmente. Em telas maiores, eles podem ser organizados em uma grade. Use cores para destacar os status: vermelho para 'críticos', amarelo para 'estoque baixo'. A tabela de produtos recentes deve ser horizontalmente rolável em mobile para não quebrar o layout

4 - Cadastrar produtos
4.1 - 
Desenvolva a funcionalidade de cadastro de novos produtos. Crie um formulário que colete nome, descrição, quantidade, quantidade mínima e preço. Implemente a lógica no backend para salvar esses dados no banco de dados, com validação dos campos obrigatórios.
